# 种草监控详情页 - 账号明细 B2 方案落地

## 用户原始诉求
- "明细有些问题，每个平台对应的账号可能是不一样的，你有什么方案"
- 决策：**B2 = 模型拆账号 + 详情页"矩阵 → 平台"二级分组**

## 落地拆解

### 1. Mock 数据模型重构（assets/js/mock/seed-monitor.js）
- 取消账号上的 `platforms: []` 多平台数组
- 改为 **(matrixId × platform) 各自独立的账号实体**：accountId 唯一对应一个平台
- 例：原来 1 个 KOC "体验官" 跨 [小红书, 抖音] → 拆成 2 条
  - `{ id: 'koc-001-xhs', matrixId: 'koc', platform: 'xhs', nickname: '体验官-红薯号' }`
  - `{ id: 'koc-001-douyin', matrixId: 'koc', platform: 'douyin', nickname: '体验官-抖音号' }`
- 员工号本就一店一平台，沿用现有"绑酒店"逻辑，每个员工号 platform 字段也唯一
- 昵称生成：在原昵称尾部追加平台短标识（小红书→红薯号 / 抖音→抖音号 / 星球号→星球号 / 视频号→视频号）
- `SEED_MONITOR_VERSION` 升 v4 → v5

### 2. 发布数据生成微调
- `buildSeedPostDaily` 内 `acc.platforms.forEach(...)` → 改为 `cells.push({ accId, acc, platform: acc.platform })` 单平台
- 总条数 / 频次模式不变（保证默认达标不破）
- hotelId 分配规则不变

### 3. 聚合函数（assets/js/utils.js）
- `aggregateSeedHotelDetail` 中 `byMatrix.accounts` 改为 `byMatrix.byPlatform[platform].accounts`
- 输出结构：
  ```
  byMatrix: [
    {
      matrixId, label, freqDesc, engagementThreshold,
      total: { posts, exposure, interact, engRate, engStatus },
      byPlatform: [
        {
          platform: 'xhs', platformLabel: '小红书',
          subtotal: { posts, exposure, interact, engRate, engStatus },
          accounts: [{ accountId, nickname, posts, exposure, interact, engRate, engStatus }]
        },
        ...
      ]
    }
  ]
  ```
- `aggregateSeedAccountsByMatrixPlatform` 也同步：账号侧入参从 `acc.platforms` 改为 `acc.platform`

### 4. 详情页（pages/ops-seed-monitor-detail.html）
- "账号明细" 区块结构：
  ```
  KOC （矩阵汇总：N 账号·M 篇·互动率 X%）
    ─ 小红书 （N 账号·M 篇·互动率 X%）
       账号  已发  曝光  互动  互动率
       ...
    ─ 抖音 （...）
       ...
  营销号
    ─ ...
  ```
- 平台子分组只显示**本酒店有发布的平台**（无发布的平台不渲染，避免空表）
- 矩阵 sub-section 头部沿用现有 `sm-matrix-section`，新增 `sm-platform-sub` 子分组样式

### 5. CSS（assets/css/pages/ops.css）
- 新增 `.sm-platform-sub`、`.sm-platform-sub__head`、`.sm-platform-sub__name`、`.sm-platform-sub__meta`
- 视觉上比 matrix 头部更轻：左侧 2px 平台色细条 + 灰底文字

## 已默认决策（如需调整请打断）
- 昵称后缀风格：`-红薯号 / -抖音号 / -星球号 / -视频号`（如想用 `(小红书)` 之类括号也可）
